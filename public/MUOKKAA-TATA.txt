AVOIMET PORTIT – SISÄLLÖN MUOKKAUS (ASIAKAS)
==========================================

1) TEKSTIT JA LINKIT
   Avaa tiedosto: site-content.json
   - "languages" → suomi (fi), ruotsi (sv), englanti (en). Muokkaa otsikoita ja kappaleita.
   - "site" → sivun otsikko ja kuvaus (hakukoneet).
   - "logo" → yläpalkin ja alatunnisteen nimi.
   - "contact" → yhteyshenkilö, puhelin, sähköposti, osoite.
   - "links" → painikkeiden ja alalaidan linkkien osoitteet.
   - "prices" → hintarivien eurot (esim. "15 €").
   - "analytics" → kävijätilastot (valinnainen). Tyhjä = ei seurantaa. Käytä vain yhtä palvelua.
     - "googleMeasurementId": Google Analytics 4 -mittaus-ID (esim. G-XXXXXXXXXX).
       Raportit: https://analytics.google.com (ilmainen Google-tili).
     - "plausibleDomain": Plausible (yksinkertaisempi, EU-ystävällinen). Esim. kevinsunde.github.io
       Raportit: https://plausible.io — vaatii tilauksen.
     - "requireConsent": true = evästepalkki ennen GA:ta (suositus). false = lataa heti.
       Plausiblella voi käyttää false, jos et käytä Google Analyticsia.

   GOOGLE ANALYTICS 4 — ASENNUSOHJE:
   1) Avaa https://analytics.google.com ja kirjaudu Google-tilillä.
   2) Admin → Luo tili / Luo ominaisuus (Property) → Verkkovirta (Web).
   3) Sivuston URL: GitHub-sivu (esim. https://kevinsunde.github.io/avoimetportit/)
      tai oma domain, jos sellainen on käytössä.
   4) Kopioi Mittaus-ID (alkaa G-).
   5) Liitä site-content.json → "googleMeasurementId": "G-XXXXXXXXXX"
   6) Työnnä muutokset GitHubiin (npm run build + push) — seuranta alkaa muutaman tunnin sisällä.

   PLAUSIBLE — ASENNUSOHJE:
   1) Luo tili osoitteessa https://plausible.io
   2) Lisää sivusto (domain = sama kuin sivuston osoite, esim. kevinsunde.github.io)
   3) Liitä "plausibleDomain": "kevinsunde.github.io" (ilman https://)
   4) Jätä googleMeasurementId tyhjäksi.
   - "registration" → ilmoittautumislomakkeen vastaanottajat:
     - "receiverEmail" ja "receiverEmail2": kaksi eri osoitetta; sama lomaketäyttö lähetetään molempiin
       (FormSubmit). Tyhjä "receiverEmail2" poistaa toisen kopion. Molemmat osoitteet on vahvistettava
       kerran palvelussa (formsubmit.co), kun lomake lähetetään ensimmäisen kerran.
     - "formAction": valinnainen oma lomake-URL. Jos tämä on asetettu, lähetys menee vain sinne yhteen
       osoitteeseen (ei kahteen); silloin kopion lähetys tehdään palvelun puolella.

   Tiedoston pitää olla kelvollista JSON:ia (pilkuilla ja lainausmerkeillä). Voit tarkistaa
   esim. osoitteessa jsonlint.com.

2) KUVAT
   - Lisää uudet kuvat kansioon: public/images/
   - Päivitä site-content.json → "images":
     - "hero": tiedostonimi, esim. "oma-kuva.jpg"
     - "gallery": lista; "wide": true tekee kuvasta leveän (ensimmäinen rivi).

   Käytä kevyitä .jpg tai .webp -kuvia, jotta sivu pysyy nopeana.

3) AJANKOHTASET-SIVU
   Tiedosto: ajankohtaset.html (valikon linkki). Tekstit: site-content.json → languages →
   avaimet ajankohtaset.* (fi, sv, en).
   Vuosikuvat: sama tiedosto → ajankohtasetByYear: lista vuosista, jokaisella "year"
   (esim. "2024") ja "images"-lista. Valinnainen "yearBanner": leveä tapahtumakuva
   (esim. vuoden logo/päivämäärä) näytetään kyseisen vuoden otsikon yläpuolella.
   Kuva voi olla pelkkä tiedostonimi tai objekti:
   { "file": "kuva.jpg", "caption": { "fi": "...", "sv": "...", "en": "..." } }.
   Etusivun yläbanner: site-content.json → images → "heroBanner".
   Kuvatiedostot kansioon public/images/.

4) JULKAISU
   Muutosten jälkeen: npm run build ja lähetä "dist"-kansion sisältö palvelimelle,
   TAI jos käytät Netlifya tms., työnnä muutokset Git-repositorioon.

5) OLETUKSET UUDELLEEN
   Kehittäjä voi luoda site-content.json:n uudelleen koodista komennolla:
   npm run content:export
